# 転職SEOメディア構築プロジェクト
テクニカルSEO設計書

> 作成者ペルソナ：Technical SEO Architect / 検索エンジンインフラコンサルタント / Webパフォーマンスエンジニア

本サイトは **Next.js + Vercel** を前提として構築する。

---

## 1. テクニカルSEO設計の目的

1. 検索エンジンがサイト構造を理解しやすくする
2. クロール効率を高める
3. インデックス品質を向上させる
4. 検索ランキングの基盤を作る

---

## 2. URL設計

| 原則 |
|------|
| 短い |
| 意味がある |
| 階層構造が明確 |
| 英語・ハイフン区切り |

| URL例 | ページ |
|-------|-------|
| `/agent` | 転職エージェントカテゴリ |
| `/agent/comparison` | 転職エージェント比較 |
| `/it` | IT転職カテゴリ |
| `/it/agent` | IT転職エージェント |
| `/second-career` | 第二新卒カテゴリ |
| `/sales` | 営業転職カテゴリ |
| `/executive` | ハイクラス転職カテゴリ |

---

## 3. titleタグ設計

| 項目 | 内容 |
|------|------|
| 文字数 | 30〜35文字 |

**例：**
- 転職エージェントおすすめ10選
- IT転職エージェント比較
- 第二新卒転職おすすめ

---

## 4. meta description設計

| 項目 | 内容 |
|------|------|
| 文字数 | 120〜160文字 |
| 内容 | 記事要約 / メリット / クリック誘導 |

---

## 5. 見出し構造

| 見出し | 用途 |
|--------|------|
| H1 | 記事タイトル（1ページ1つ） |
| H2 | 主要セクション |
| H3 | サブセクション |
| H4 | 補足情報 |

---

## 6. パンくず設計

```
例：TOP > IT転職 > IT転職エージェントおすすめ
```

構造化データ：`BreadcrumbList` を使用

---

## 7. 内部リンク設計

```
カテゴリページ → ピラー記事 → クラスター記事
```

加えて「関連記事」「人気記事」リンクを配置する。

---

## 8. sitemap.xml

自動生成。含めるURL：

| 対象 |
|------|
| TOPページ |
| カテゴリページ |
| 記事ページ |
| 比較ページ |

---

## 9. robots.txt

| 設定 | 対象 |
|------|------|
| クロール許可 | 記事ページ / カテゴリページ |
| クロール除外 | 管理ページ / テストページ |

---

## 10. canonical設計

重複ページ対策として `canonical` タグを全ページに設定する。

---

## 11. 構造化データ

| Schema | 用途 |
|--------|------|
| Article | 記事ページ |
| FAQPage | FAQ部分 |
| BreadcrumbList | パンくず |
| Organization | サイト全体 |
| ItemList | ランキング |

---

## 12. 表示速度最適化（Core Web Vitals）

| 指標 | 目標値 |
|------|--------|
| LCP | 2.5秒以内 |
| CLS | 0.1以下 |
| INP | 200ms以下 |

---

## 13. 画像最適化

| 項目 | 内容 |
|------|------|
| フォーマット | WebP |
| 最適化 | Next.js Image Optimization |
| 読み込み | Lazy Load |

---

## 14. モバイルSEO

| 項目 | 内容 |
|------|------|
| 設計方針 | モバイルファースト |
| レスポンシブ | 対応 |
| タップ領域 | 48px以上 |

---

## 15. インデックス管理

| 設定 | 対象 |
|------|------|
| index | 記事 / カテゴリ / 比較ページ |
| noindex | テストページ / 未完成ページ |

---

## 16. ページネーション

記事一覧ページは `/page/2` 形式のページネーションを使用する。

---

## 17. AMP

**採用しない。** Next.js高速表示 + Core Web Vitals対応で十分なため。
