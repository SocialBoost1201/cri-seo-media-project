# 転職SEOメディア構築プロジェクト
GEO（Generative Engine Optimization）設計書

> 作成者ペルソナ：Generative Search Strategist / AI検索最適化アーキテクト / 検索アルゴリズム戦略コンサルタント

GEO（Generative Engine Optimization）は従来のSEOを拡張した概念であり、
「検索エンジン」「AI検索」「ナレッジ回答」のすべてに最適化する設計を指す。

---

## 1. GEO戦略の目的

1. AI検索に引用される情報構造を作る
2. AI回答の情報ソースとして選ばれる
3. 検索結果のゼロクリック化に対応する
4. SEOコンテンツをAI検索資産化する

---

## 2. AI検索が優先引用する情報

AI検索（ChatGPT / Perplexity / Gemini）は以下の情報を優先して引用する。

| 情報タイプ | 概要 |
|-----------|------|
| 構造化された情報 | 表・リスト・見出し構造 |
| 比較表 | 複数サービスの並列比較 |
| 定義コンテンツ | 「〇〇とは」形式 |
| FAQ | 質問と回答が明確なもの |
| 統計データ | 数値による裏付け |
| 明確な結論 | 冒頭に結論を置く構造 |

---

## 3. GEO最適化コンテンツ構造

すべての記事は以下の構造で設計する。この構造はSEO・AI検索の両方に最適化されている。

```
タイトル
↓
結論（要約）
↓
比較表
↓
ランキング
↓
詳細説明
↓
FAQ
↓
まとめ
```

---

## 4. 定義コンテンツ設計

AI検索は「〇〇とは」形式の定義コンテンツを頻繁に引用する。
各記事内に以下の構造で **定義セクション** を設置する。

```
定義
↓
特徴
↓
メリット
↓
デメリット
```

**例：**
- 転職エージェントとは
- 第二新卒とは
- IT転職とは

AI検索では比較情報が引用されやすい。
以下の項目で比較表を設置する。

| 比較項目 |
|---------|

---

## 6. FAQ構造

AI検索はFAQを回答として引用する。各記事に **5〜10項目** のFAQを設置する。

**FAQ例：**
- 転職エージェントは複数登録すべきか
- 転職エージェントは無料か
- 第二新卒でも転職できるか
- 営業職は転職しやすいか

---

## 7. 要約コンテンツ（冒頭結論）

AI検索では記事全体の要約が重要。記事冒頭に「**結論**」を設置する。

```
例：おすすめ転職エージェントは以下です。
1. 〇〇エージェント
2. 〇〇エージェント
3. 〇〇エージェント
```

---

## 8. ランキング構造

AI検索はランキング情報を引用する。ランキングは **比較基準を明確に** 設定する。

```
例：おすすめ転職エージェント
1位：〇〇エージェント
2位：〇〇エージェント
3位：〇〇エージェント
```

## 10. AI引用率向上の設計方針

AI引用率が高い記事の共通点：

| 要素 | 設計方針 |
|------|---------|
| 明確な結論 | 記事冒頭に配置 |

## 14. GEOとSEOの関係

GEOはSEOの代替ではなく **拡張** である。

| 施策 | 目的 |
|------|------|
| SEO | 検索順位の向上 |
| GEO | AI回答への掲載 |

両方を最適化することで、次世代検索環境に対応する。

---

## 15. 本プロジェクトにおけるGEOの位置付け

SEO・GEOの両軸で検索流入を獲得することで、
Google検索・AI検索の双方からのユーザー獲得を実現する。

```
Google検索 ──→ SEO最適化記事
AI検索    ──→ GEO最適化コンテンツ
              ↓
        転職情報メディア
              ↓
           Webist送客
```
